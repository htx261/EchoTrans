# AI 同声传译助手设计文档

## 1. 项目目标

开发一款 Windows 桌面端 AI 同声传译助手，帮助用户观看英语或其他外语演讲、技术分享、国际会议、网课时降低语言门槛，提高信息获取效率。

第一版聚焦“实时中文字幕”，暂不包含中文语音播报、用户账号、收费授权、在线翻译 API。

核心目标：

- 优先支持本地音视频文件播放，同时为后续网络流/直播 URL 预留结构。
- 从播放中的媒体提取单向音频流。
- 使用 `whisper.cpp` 自动检测音频语言并转录原文。
- 使用 CTranslate2 加载本地翻译模型，将原文翻译为简体中文。
- 在视频上覆盖显示当前字幕，并在侧边栏显示双语字幕历史。
- 通过“临时字幕 + 确认字幕”的机制支持自动修正。
- 支持导出双语 `.srt` 字幕文件。

## 2. 已确认技术选型

- 开发语言：C++
- UI 框架：Qt Widgets
- 开发环境：Visual Studio
- 构建系统：CMake
- 第一版目标平台：Windows
- 播放器后端：FFmpeg
- 语音识别：`whisper.cpp`
- 本地翻译推理：CTranslate2 C++ API
- 翻译方向：多语言原文到简体中文
- 模型分发：大模型不提交到 Git，不强制打包进安装包
- 模型加载：程序从用户配置的本地模型目录加载
- 运行方式：Qt 主程序内部直接调用 FFmpeg、`whisper.cpp`、CTranslate2 的 C/C++ API

## 3. 推荐翻译模型

第一版推荐使用 `facebook/nllb-200-distilled-600M`，转换为 CTranslate2 格式后进行量化，并以 CPU 推理作为基线。

选择原因：

- 支持多语言互译，适合“自动识别多语言原文并翻译成中文”的目标。
- 能配合 Whisper 的语言检测结果使用，不需要为每个语种维护独立的双语模型。
- 本地运行，不依赖 API Key，也不会产生在线服务费用。
- 相比更大的 NLLB 模型，distilled 600M 版本更适合第一版本地桌面应用。

翻译模块不应该完全依赖模型自行猜测源语言。更稳妥的流程是由 `whisper.cpp` 输出检测到的语言，再由翻译模块映射为 NLLB 语言码。

示例语言码映射：

- 英语：`eng_Latn`
- 日语：`jpn_Jpan`
- 韩语：`kor_Hang`
- 法语：`fra_Latn`
- 德语：`deu_Latn`
- 简体中文目标语言：`zho_Hans`

如果检测到的语言暂时没有配置映射，翻译模块应返回明确错误，并回退为显示原文字幕。

## 4. 总体架构

第一版是单体桌面应用，但内部模块边界要清晰。

```text
媒体文件 / 网络流
  -> FFmpeg 播放与音频解码
  -> 音频分段缓冲
  -> whisper.cpp 语言检测与原文转录
  -> CTranslate2 本地翻译为简体中文
  -> 字幕管理器
  -> Qt UI 字幕覆盖、历史侧栏、字幕导出
```

建议模块划分：

```text
ui/
  MainWindow
  播放器控件
  字幕覆盖层
  字幕历史侧栏
  模型设置页

media/
  FFmpeg 解封装与解码
  播放控制
  音频帧抽取
  音量与静音控制

asr/
  whisper.cpp 封装
  模型加载
  语言检测
  分段转录

translation/
  CTranslate2 封装
  翻译模型加载
  Whisper 语言码到 NLLB 语言码映射
  多语言原文到简体中文翻译

subtitle/
  临时字幕
  确认字幕
  修正历史
  双语 SRT 导出

core/
  任务调度
  工作线程
  配置管理
  错误上报
```

第一版不通过命令行拼接调用 ASR 或翻译工具，而是在主程序内直接调用本地 C++ API。接口设计仍需保留未来拆分为 worker 进程的可能性。

## 5. 界面设计

界面采用“传统播放器 + 字幕学习侧栏”的组合。

主窗口包含：

- 中央视频/播放器区域
- 视频上方当前中文字幕覆盖层
- 底部播放控制条
- 右侧字幕历史侧栏
- 模型与路径设置窗口

播放控制包含：

- 打开本地媒体文件
- 播放/暂停
- 进度跳转
- 音量调节
- 静音

字幕显示包含：

- 视频覆盖层显示最新中文字幕。
- 右侧历史区显示原文、中文翻译、时间戳、字幕状态。
- 临时字幕和确认字幕要有视觉区分。
- 确认字幕稳定后进入历史记录，并用于导出。

第一版不做中文 TTS 语音输出。

## 6. 实时数据流

第一版采用平衡型实时策略：

- 短音频窗口先生成临时字幕，降低用户等待时间。
- 更稳定的句子级片段生成确认字幕。
- 最近的临时字幕允许被 ASR 或翻译结果替换。
- 字幕超出修正窗口后固定为确认字幕。

推荐流程：

```text
1. FFmpeg 解码媒体并播放原声。
2. 音频帧复制到 ASR 音频缓冲区。
3. ASR 工作线程处理短窗口，输出临时原文候选。
4. 字幕管理器创建或更新临时字幕。
5. 当片段稳定后，ASR 输出带语言和时间戳的确认原文。
6. 翻译工作线程将确认原文翻译为简体中文。
7. 字幕管理器用确认双语字幕替换临时字幕。
8. UI 更新视频字幕覆盖层和右侧历史侧栏。
```

延迟目标：

- 普通 CPU 笔记本上，临时字幕应在几秒内出现。
- 确认字幕可以比临时字幕略慢，但要适合网课、演讲、技术分享的跟读节奏。

## 7. 修正机制

字幕分为两种状态：

- 临时字幕：低延迟，可变化。
- 确认字幕：相对稳定，可进入历史记录和导出。

修正窗口：

- 系统可以修正当前字幕和最近几条字幕。
- 较早的确认字幕默认不再修改，除非用户重新处理媒体。

修正来源：

- Whisper 在获得更长音频上下文后可能修正识别文本。
- 翻译模块在句子边界、标点更清晰后可能得到更好的译文。
- 字幕管理器需要保持字幕条目 ID 稳定，让 UI 更新已有行，而不是不断追加重复行。

行为要求：

- 不要大范围重写整段字幕历史。
- 当前字幕可以较频繁更新。
- 历史区展示稳定后的确认文本。
- 导出默认只使用确认字幕。

## 8. 模型管理

大模型不提交到 Git 仓库。

仓库应包含：

- 模型下载说明
- 推荐模型目录结构
- 配置示例
- 缺失模型时的运行提示

建议本地目录结构：

```text
models/
  whisper/
    ggml-base.bin
    ggml-small.bin
  translation/
    nllb-200-distilled-600m-ct2-int8/
      config.json
      model.bin
      shared_vocabulary.json
      sentencepiece.bpe.model
```

程序设置页应允许用户配置：

- Whisper 模型路径
- 翻译模型目录
- 目标语言，第一版固定为简体中文
- CPU 线程数

如果模型缺失，程序应显示清晰的配置提示，而不是静默失败。

## 9. FFmpeg 播放器模块

播放器模块职责：

- 打开本地媒体文件
- 为后续网络流 URL 预留媒体源抽象
- 解封装音视频流
- 解码视频帧并显示
- 解码音频帧并播放
- 向 ASR 管线提供 PCM 音频帧
- 支持播放、暂停、跳转
- 支持原声音量调节和静音

第一版优先支持本地文件。媒体源接口仍应使用 URI/路径抽象，便于后续扩展网络流。

## 10. 线程与调度

UI 线程不能执行模型推理。

建议线程划分：

- UI 线程：Qt 控件与用户交互
- 媒体线程：FFmpeg 解码与播放协调
- ASR 工作线程：Whisper 转录
- 翻译工作线程：CTranslate2 翻译

模块间通信可使用 Qt signals/slots 或小型线程安全任务队列。

ASR 和翻译任务需要支持取消：

- 用户关闭文件
- 用户跳转进度
- 用户切换模型
- 应用退出

## 11. 配置管理

本地配置可使用 `QSettings` 保存。

第一版配置项：

- 最近打开的媒体目录
- Whisper 模型路径
- 翻译模型目录
- 音频语言模式：自动检测
- 目标语言：简体中文
- CPU 线程数
- 字幕导出目录
- 播放器音量

第一版不需要登录、收费、授权、云同步。

## 12. 字幕导出

应用支持导出双语 `.srt` 字幕。

每条字幕包含：

- 开始时间
- 结束时间
- 原文
- 中文翻译
- 源语言
- 确认状态

导出示例：

```text
1
00:00:01,200 --> 00:00:04,800
Original: Today I want to talk about distributed systems.
中文: 今天我想谈谈分布式系统。
```

默认只导出确认字幕。

## 13. 错误处理

重要错误应在 UI 中明确提示：

- 媒体打开失败
- FFmpeg 解码失败
- Whisper 模型缺失或不兼容
- 翻译模型缺失或不兼容
- 语言码映射缺失
- CPU 内存不足
- 因跳转或关闭文件导致任务取消

系统需要优雅降级：

- 翻译失败时显示原文字幕。
- ASR 未就绪时播放器仍可播放。
- 没有确认字幕时，导出功能应给出明确提示。

## 14. 测试策略

核心模块应能脱离完整 UI 单独测试。

推荐自动化测试：

- 字幕管理器状态流转：临时、确认、修正、导出。
- Whisper 语言码到 NLLB 语言码的映射。
- 配置加载与路径校验。
- SRT 导出格式。
- 本地媒体路径抽象。

推荐手动验证：

- 打开本地英语视频并播放。
- 原声播放、音量调节、静音可用。
- 临时字幕先于确认字幕出现。
- 确认字幕进入右侧历史区。
- 双语 SRT 导出可用。
- 模型路径缺失时显示清晰提示。

## 15. 第一版范围

第一版包含：

- Windows 桌面应用
- Qt Widgets UI
- 可被 Visual Studio 打开的 CMake 项目
- Git 项目管理
- FFmpeg 本地文件播放
- 原声音量与静音
- 音频帧抽取给 ASR
- `whisper.cpp` 模型加载与转录封装
- CTranslate2 翻译封装
- NLLB 语言码映射
- 临时字幕和确认字幕状态
- 当前字幕覆盖层
- 字幕历史侧栏
- 双语 SRT 导出
- 本地模型路径配置

第一版不包含：

- 中文语音播报
- 用户账号
- 收费或授权逻辑
- 在线翻译 API
- 将大模型提交到仓库
- 完整跨平台支持
- 完整网络直播流支持
- 移动端支持

## 16. 实现注意事项

- CPU-only 是第一版性能基线。
- Whisper 模型大小应可配置，小模型降低延迟，大模型提升准确率。
- 翻译任务要节流，避免对不稳定的临时文本频繁翻译导致 CPU 负载过高。
- 模型加载和推理期间 UI 必须保持响应。
- Windows 第三方二进制依赖需要有清晰文档。
- 模块接口要为未来拆分 ASR/翻译 worker 进程保留空间。
